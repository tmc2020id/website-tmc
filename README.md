# Website TMC - 3D Engine

هذا المستودع يحتوي على الكود المصدري لمشروع Website TMC. تم إعداد هذا المستودع لتطوير تجارب رسومية ثلاثية الأبعاد (3D) باستخدام `Three.js` و `Cannon-es` مع خادم تطوير `Vite` محسن ومعد خصيصاً لتطبيقات الويب عالية الأداء.

## مميزات بيئة التطوير (Vite)
- **HMR (Hot Module Replacement) سريع**: تحديثات فورية للكود بدون إعادة تحميل الصفحة.
- **Tree Shaking**: إزالة الأكواد غير المستخدمة من المكتبات الضخمة مثل `Three.js` لتقليل حجم الملف النهائي.
- **Source Maps**: تصحيح أخطاء سهل عبر متصفحك.
- **دعم Shaders**: يمكن استيراد ملفات `.glsl`, `.vert`, و `.frag` مباشرة كأكواد نصية (String).
- **Path Aliases**: استيراد الملفات بسهولة عبر `@/` بدلاً من المسارات الطويلة `../../../`.

## كيفية التشغيل

### 1. تثبيت الاعتمادات
تأكد من أنك قمت بتثبيت `Node.js` (إصدار 16 أو أحدث). ثم قم بتشغيل الأمر التالي:
```bash
npm install
```

### 2. تشغيل خادم التطوير (Local Server)
لتشغيل الخادم المحلي على المنفذ `5173`:
```bash
npm run dev
```
سيعمل الخادم على الرابط: [http://localhost:5173/](http://localhost:5173/)

### 3. بناء المشروع للإنتاج (Production Build)
لإنشاء نسخة محسنة ومضغوطة من المشروع لرفعها على الاستضافة:
```bash
npm run build
```

## هيكل المجلدات

- `src/` : يحتوي على الكود المصدري الرئيسي.
  - `src/assets/` : للملفات الثابتة.
    - `src/assets/models/` : لمجسمات 3D (مثل GLTF/GLB).
    - `src/assets/textures/` : للصور وخامات المجسمات.
  - `src/shaders/` : لملفات الـ GLSL (Vertex & Fragment Shaders).
  - `src/components/` : لملفات أو فئات (Classes) واجهة المستخدم.
  - `src/main.ts` : نقطة الدخول الرئيسية لتهيئة الـ 3D والفيزياء.

## إجراءات تصحيح الأخطاء الشائعة (Troubleshooting)

1. **مشكلة CORS (Cross-Origin Resource Sharing) عند تحميل الـ Models أو Textures:**
   - **السبب:** يحدث عند محاولة تحميل ملفات من مسارات خارج بيئة السيرفر المحلي.
   - **الحل:** تم ضبط إعدادات Vite للسماح بـ CORS محلياً، تأكد دائماً من وضع الملفات في مجلد `public/` أو استيرادها عبر `import` إذا كانت في مجلد `src/assets/`.

2. **خطأ "Module not found" عند استيراد ملف Shader:**
   - **الحل:** تأكد من أنك قمت بإضافة اللاحقة `.glsl` أو أنك قمت بتحديث ملف `vite-env.d.ts` لدعم نوع الملف. الإضافة `vite-plugin-glsl` مهيأة بالفعل في `vite.config.ts`.

3. **المنفذ 5173 مشغول:**
   - **الحل:** السيرفر سيبحث تلقائياً عن منفذ بديل. يمكنك تغييره يدوياً عن طريق إنشاء ملف `.env` وإضافة `PORT=3000` أو تعديل `vite.config.ts`.

4. **بطء في الأداء أثناء التطوير:**
   - **الحل:** إذا كان المشهد يحتوي على مجسمات ضخمة جداً، حاول تقليل دقة الـ Shadows، أو تعطيل الـ Antialiasing في `THREE.WebGLRenderer` أثناء فترة التطوير.

# Website TMC

هذا المستودع يحتوي على الكود المصدري لمشروع Website TMC. تم إعداد هذا المستودع لتتبع تاريخ التغييرات البرمجية والتعاون بين المطورين.

## كيفية استخدام المستودع

### استنساخ المستودع (Clone)
للبدء في العمل على المشروع، قم باستنساخ المستودع إلى جهازك المحلي:
```bash
git clone <رابط_المستودع>
cd website-tmc
```

### الفروع (Branches)
الفرع الرئيسي للمشروع هو `main`. يجب أن يكون هذا الفرع مستقراً وجاهزاً للإنتاج دائماً.
- **main**: الفرع الرئيسي الذي يحتوي على الكود المستقر.
- **feature/branch-name**: للفروع التي يتم فيها إضافة ميزات جديدة.
- **bugfix/branch-name**: للفروع المخصصة لإصلاح الأخطاء.

### أسس المساهمة (Contribution Guidelines)

1. قم بإنشاء فرع جديد للميزة أو الإصلاح الذي تعمل عليه:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. قم بإجراء التعديلات المطلوبة، وتأكد من اختبارها.

3. اكتب رسائل Commit واضحة ووصفية تتبع النمط التالي:
   `[نوع_التغيير]: [وصف_مختصر]`
   
   أنواع التغييرات المعتمدة:
   - `feat`: إضافة ميزة جديدة.
   - `fix`: إصلاح خطأ برمجي.
   - `docs`: تعديلات في التوثيق.
   - `style`: تغييرات في التنسيق (مسافات، فواصل، إلخ).
   - `refactor`: إعادة هيكلة الكود دون إضافة ميزة أو إصلاح خطأ.
   - `test`: إضافة أو تعديل الاختبارات.
   - `chore`: تحديثات في أدوات البناء، المكتبات، وغيرها.

   **مثال:** `feat: إضافة صفحة تسجيل الدخول`

4. ارفع الفرع الخاص بك إلى المستودع البعيد:
   ```bash
   git push origin feature/your-feature-name
   ```

5. قم بإنشاء Pull Request (PR) لمراجعة الكود ودمجه في الفرع `main`.

## قواعد حماية الفرع الرئيسي (Branch Protection)
الفرع `main` محمي بقواعد تمنع التعديل المباشر (Direct Commits).
- لا يمكن الدفع المباشر (Push) للفرع `main`.
- يجب دمج جميع التغييرات عبر Pull Requests فقط.
- يُفضل أن يمر الـ PR بمراجعة (Code Review) قبل الدمج.
